Core Page Rhythm

The case studies should feel like one system, not three templates.

Every major section should start with the same open structure: eyebrow, large heading, divider, then content. This is the strongest pattern in Car & Classic and should be the default unless a section is a quote or a hero.

Cards should carry individual ideas, not entire sections. If a heading is inside a card while surrounding sections use open headings, the page starts to feel assembled from different systems.

01

Open section header

Use for every main narrative section. Eyebrow, section heading, divider, then content. This should be the primary section type across all case studies.

Keep as default

02

Two column copy block

Use when a section needs a concise setup and supporting explanation. Strong for context, research findings, and outcomes.

Keep

03

Context pair

Use for Product Context and Design Principle. One dark card and one light card. Avoid using it as decoration.

Keep, use sparingly

04

Big quote moment

Use once per case study for the line that changes the frame. It should be short, specific, and attributed simply.

Keep

Evidence Modules

Research and process artefacts need fewer styles and clearer roles.

Recommended

Side by side evidence block

Use when one image supports one idea. Text and image should align at the title, not the vertical middle. This works for data models, workflow maps, IA comparisons, and key process artefacts.

Use for Brain in Hand process blocks

What Works

The artefact has a clear caption and sits directly next to the point it proves.

Out Of Alignment

A gallery of process images appears without explaining what the reader should learn from them.

01
Research

Stats grid

Use for quantitative context. Four numbers maximum. The numbers should explain the scale or constraint, not decorate the section.

02
Process

Process cards

Use for named process stages. Each card should have one label, one claim, one image or short body.

03
Reflection

Principle cards

Use for reflections and design decisions. Match the Car & Classic card treatment wherever possible.

Media Patterns

Images should either explain the story or carry the result.

Large artefact plus explanation

Use when a single artefact is important enough to slow the reader down. This is strongest when the caption labels the artefact and the adjacent copy explains why it mattered.

A

Final screens

Use stacked image cards or a scrolling flow. Captions should be written for readers, not copied from file names.

Keep

B

Handoff media

Use when the artefact proves senior design practice. Prototype video, anatomy docs, component specs, and spacing systems belong near handoff.

Keep in Handoff

Consistency Rules

A simple checklist before a case study is called finished.

01

One section header system

Main sections use eyebrow, heading, divider. Avoid placing the section title inside a card unless every comparable section does the same.

02

One caption voice

Captions should explain the image in portfolio language. Avoid raw file names, casual internal labels, and unexplained acronyms.

03

One card family

Decision cards, reflection cards, and principle cards should share the same visual family unless they serve clearly different roles.

04

Every image has a job

If an image does not prove, explain, or clarify a point, remove it or move it to a supporting gallery.
